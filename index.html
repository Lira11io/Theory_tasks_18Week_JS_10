Привет, Кот. Вот мои ответы на теоретические вопросы 18 недели.

  1. Сколько живет localStorage?

    Ответ: localStorage "живет" бессрочно, до тех пор, пока данные не удалят вручную.

  2. Как посмотреть localStorage в браузере?

    Ответ: Посмотреть localStorage также, как и остальные хранилища можно в инструментах разработчика во вкладке Application.

  3. Можно ли хранить данные пользователя в sessionStorage?

    Ответ: хранить на постоянной основе не получиться, так как sessionStorage хранит данные в течение продолжения текущей сессии.
    И после закрытия браузера, данные становятся недоступными.

  4. Как очистить локальное хранилище в браузере?

    Ответ: для этого есть метод clear().

  5. Для чего нужны куки JS?

    Ответ: cookie позволяют хранить и пользоваться данными пользователя и в браузере, и на сервере.

  6. В чём польза cookie?

    Ответ: хранимые в них данные помогают браузеру распознать сайт при повторном посещении и выполнить определенные действия,
    например, в виде автоматической авторизации. Благодаря файлам куки веб-браузер на компьютере или мобильном устройстве обменивается
    с сервером сайта персональной информацией пользователя.
    Основные функции куки-файлов:
    Быстрая авторизация. Браузер при регистрации сохраняет ваши логин и пароль для простого повторного посещения ресурса — не надо каждый раз вводить
    их снова. Перед сохранением данных браузер спрашивает разрешение.
    Сохранение настроек сайта. Браузер запомнит выбранный вами дизайн, язык, фильтрацию, вашу локацию и так далее.
    Мониторинг шагов. Ваши действия на сайте фиксируются для статистики и для маркетингового анализа, то есть используются как действия потенциального
    клиента для выдвижения рабочих теорий по контент-маркетингу.

  7. Как работать с cookie?

    Ответ: Все куки хранятся в свойстве document.cookie. Это свойство представляет собой строку в формате имя=значение, где пары имён и значений разделяются
    знаком ; . При этом взаимодействие с полем весьма необычное — если присвоить document.cookie новое значение, то оно не заменит полностью старую строку,
    а добавит или изменит значение по ключу.
    Запись в cookie работает с помощью присвоения значения новой куки в поле document.cookie. За один раз можно записать лишь одно значение.
    При присвоении свойству куки с другим именем, получим два записанных значения.
    При повторной записи в то же поле другого значения оно будет перезаписано.
    При установке кук можно указывать не только её название и значение, но и другие параметры. Все они являются необязательными и разделяются точкой
    с запятой ; (path; domain; max-age и expires; secure; samesite = strict или samesite = lax).
    Для получения значений, записанных в куки, можно просто вывести содержимое document.cookie: console.log(document.cookie);
    Для кук не предусмотрено специального метода удаления, поэтому для этого используется трюк с установкой кук с параметром expires который указывает на
    дату в прошлом. Браузер сразу же считает такую куку устаревшей и удаляет её

  8. Как добавить и получить значение из веб-хранилища?

    Ответ: для этого есть методы setItem('ключ', 'значение') и getItem('ключ').
    setItem позволяет добавить новую запись с ключом (названием), по которому будет сохранено значение, и само значение.
    Например, window.localStorage.setItem('name', 'Вася');
    getItem отвечает за чтение. С помощью ключа возвращаем полученное значение из хранилища.
    Например, window.localStorage.getItem('name');

  9. Придумайте ещё минимум 3 ситуации, помимо предложенных в уроке, для чего может быть нужно сохранять данные пользователя и какие?
    *Пример из урока: запомнить, что пользователь уже залогинился, что у него лежит в корзине или в каком разделе сайта он сейчас находится*

    Ответ: браузер запомнит выбранный вами дизайн, язык, фильтрацию, вашу локацию. Если вы просматривали какую-то информацию и случайно закрыли сайт.
    То при дальнейшей работе, вам будут предлагаться ссылки на похожие сайты. Иногда можно загрузить в плеер из базы данных какие-то песни, при последующем включении, они автоматически будут доступны.

  10. Какие преимущества использования JSON?

    Ответ:

    • JSON — это компактный формат. С ним большие объемы данных быстро обмениваются между браузером и веб-сервером.
    • С JSON-файлами можно работать не только методами JavaScript. Почти у всех языков есть инструменты для чтения и генерации данных JSON.
    • Хранение и экспорт данных в JSON поддерживают современные реляционные базы данных, такие как PostgreSQL и MySQL.

  11. Какие значения могут быть использованы в JSON?

    Ответ: строка JSON состоит из набора пар ключ-значения. В этой паре ключ отделяется от значения с помощью знака двоеточия :, а одна пара от другой
    с помощью запятой ,. Ключом может быть только строка, обёрнутая в двойные кавычки. А вот значениями ключей могут быть: массивы, объекты, строки, числа, булевый тип, значение null.

  12. Как брать данные из JSON?

    Ответ: JS позволяет работать с форматом JSON через 2 специальные команды:
    Сериализация - JSON.stringify() — превращает объект в JSON-строку для последующей отправки на сервер; Десериализация - JSON.parse() — выполняет обратное действие, превращая JSON-данные в объекты или переменные.
    Если нам нужно отправить какие-то данные на сервер, то мы должны сначала упаковать их в строку JSON-формата (сериализовать) и только потом отправлять. Метод JSON.stringify делает из объекта строку JSON.
    Для обратной операции — десериализации или парсинга json-объекта в javascript применяется метод JSON.parse(). Он преобразовывает JSON-строку в объект JavaScript.
